# Zomato-Restaurant-Analysis
A comprehensive data science project analyzing Zomato's restaurant metadata and user reviews. Features sentiment analysis via NLP and restaurant clustering using Unsupervised Machine Learning to derive actionable business insights.


Zomato Restaurant Clustering & Sentiment Analysis
Project Overview
This project leverages Zomato's rich dataset to explore the dynamics of the Indian food industry. By merging restaurant metadata with extensive user reviews, we aim to simplify the dining discovery process for customers and provide strategic, data-driven insights for the company to improve service quality and platform engagement.
Key Objectives
•	Exploratory Data Analysis (EDA): Understand the relationship between restaurant costs, ratings, and customer sentiment.
•	Sentiment Analysis (NLP): Develop an automated pipeline to categorize millions of unstructured reviews into Positive and Negative sentiments.
•	Restaurant Clustering: Use Unsupervised Machine Learning (K-Means) to group restaurants into distinct business segments.
•	Actionable Strategy: Devise data-backed strategies for Zomato to promote high-performing clusters and intervene in underperforming ones.
Tech Stack
•	Languages: Python
•	Libraries: Pandas, NumPy, Matplotlib, Seaborn, Scikit-learn, NLTK, TextBlob
•	Techniques: TF-IDF Vectorization, Sentiment Analysis, Logistic Regression, Random Forest, K-Means Clustering.
Project Structure
	1.	Data Wrangling: Cleaning messy metadata, handling missing values, and outlier treatment.
	2.	Feature Engineering: NLP pipeline (Tokenization, Lemmatization, TF-IDF).
	3.	Machine Learning: Sentiment Classification and Clustering implementation.
	4.	Deployment: Model serialization using joblib for production-ready inference.
